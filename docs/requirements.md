タイマッサージ店 Webサイト 要件仕様書（ドラフト）
1. 目的

タイマッサージ店の魅力（高級感・安心感）を伝え、予約につなげることを目的とする。

スマホ閲覧を主軸にしつつ、PCでも豪華に見えるデザインにする。

2. 想定ターゲット

20〜60代の来店客（仕事疲れ、観光、リラクゼーション目的）

予約前に「料金・アクセス・スタッフ・雰囲気」が気になる人

3. サイト全体要件
3.1 デザイン方針

基本色：緑系（深いグリーン寄り）

高級感：深緑＋ゴールド/アイボリーのアクセント想定

“豪華”の表現

余白を広く、タイポグラフィを上品に

背景に薄い柄（タイ風の紋様/ボタニカル）を控えめに

写真（店内/施術イメージ）を大きく使う

共通コンポーネント

**全ページ共通ヘッダーにメニュー（ナビ）**を設置

フッターに店舗情報・注意事項・SNS（あれば）・コピーライト

3.2 重要な文言（必須表示）

「お支払いは現金のみ」 を明確に表示

Topページの目立つ場所（例：ヒーロー直下のバッジ/注意書き）

料金ページにも再掲

予約ページにも再掲（最終確認として）

3.3 ページ構成（必須）

Top（トップページ）

Access（アクセス：Google Map 埋め込み）

Pricing（料金体系）

Staff（スタッフ紹介）

Reservation（予約ページ：LP形式）

3.4 画像の扱い（必須）

生成/用意した画像は /image フォルダに格納すること

例：/image/hero.jpg, /image/interior-01.jpg, /image/staff-01.jpg

ページ内参照は /image/... を使用

4. 画面・機能要件（ページ別）
4.1 Top（トップページ）
目的

店の雰囲気（豪華感）を伝え、予約ページへ誘導する。

セクション構成（例）

ヒーロー

大きな背景画像（店内や施術イメージ）

キャッチコピー（例）

「深い癒しを、上質な空間で。」

サブコピー（例）

「タイ伝統の手技で、心身を整えるリラクゼーション」

バッジ/注意：**「現金のみ」**を視認性高く表示

予約導線（必須）

**「Topページから予約ページへのボタンをヒーローの下くらい」**に配置

CTA例：「今すぐ予約する」

スクロールしても迷わないよう、上部にも小CTAがあっても良い

施術メニュー簡易紹介（4種）

フット / タイ / オイル / ヘッドスパ をCard UIで

店舗の強み（3点程度）

例：個室、技術、清潔感、アクセス良好 など

お客様の声（ダミーでOK）

2〜3件

フッター

店舗名、住所、営業時間、電話、現金のみ注意、コピーライト

4.2 Access（アクセス：Google Map付き）
必須要素

店舗住所（ダミーでOK。後で置換できるように）

最寄駅からの案内（徒歩○分）

営業時間

電話番号（ダミーOK）

Google Map 埋め込み（iframe）

ピンの場所は仮でもOK（後で差し替え可能な設計に）

UI

Mapはファーストビュー付近に大きめに配置

スマホでは縦に並ぶレイアウト

4.3 Pricing（料金体系）
必須：メニュー種別

フットマッサージ

タイマッサージ

オイルマッサージ

ヘッドスパ

表示形式

各メニューごとにCard UI or テーブル（豪華感が出る方）

**時間と料金（サンプル）**を表示（※あとで変更しやすい構造）

サンプル料金（案）

フットマッサージ

30分 3,500円 / 60分 6,500円

タイマッサージ

60分 7,500円 / 90分 10,500円 / 120分 13,500円

オイルマッサージ

60分 9,000円 / 90分 12,500円 / 120分 16,000円

ヘッドスパ

30分 4,000円 / 60分 7,000円

注意事項（必須）

「お支払いは現金のみ」（強調）

施術前後の注意（例：体調、遅刻時の対応など。ダミーでもOK）

4.4 Staff（スタッフ紹介）
目的

安心感・上質感（信頼）を作る。

必須要素

スタッフ一覧（3名程度のダミーデータ）

写真（/image/staff-01.jpg など）

名前（例：Aki / Nana / Mai）

得意施術（例：タイ/オイル）

一言（短いプロフィール）

予約導線（「このスタッフを指名して予約」などは将来拡張。現時点は通常CTAでOK）

4.5 Reservation（予約ページ：LP形式）
目的

迷わず予約完了に導く 1枚完結のLP。

構成（例）

ファーストビュー（強いCTA）

ヒーロー＋「予約する」ボタン（フォームへスクロール）

現金のみ注意を再掲

選ばれる理由（3つ）

技術 / 空間 / 清潔さ 等

予約の流れ（3ステップ）

①希望メニュー選択 → ②日時入力 → ③連絡先送信

予約フォーム（必須）

入力項目（例）

お名前（必須）

電話番号 or メール（必須）

希望メニュー（必須：4種）

希望時間（必須：例 30/60/90/120）

希望日時（必須：日付・時間）

指名（任意）

備考（任意）

送信後の表示

「送信しました（ダミー）」でも可（実装方針による）

少なくとも UI 上の完了メッセージは必要

FAQ（3〜5個）

例：「現金以外は使えますか？」→ 現金のみ を明確化

例：「遅刻した場合は？」など

最後のCTA

「予約する」ボタン（フォームへ戻る）

5. 共通UI要件
5.1 ヘッダー（全ページ共通）

左：ロゴ（テキストでもOK）

右：メニュー

Top / Access / Pricing / Staff / Reservation

スマホ：ハンバーガーメニュー

Reservation はCTA風に目立たせる（ボタンスタイル）

5.2 フッター（全ページ共通）

店舗情報（住所、営業時間、電話）

注意書き：現金のみ

コピーライト

6. 非機能要件（最低限）

レスポンシブ対応（スマホ優先）

表示速度：画像は適度に圧縮、遅延読み込みが望ましい

アクセシビリティ

画像に alt

ボタンやリンクが識別しやすい

SEO（簡易）

各ページに title / description

見出し構造（h1→h2→h3）を崩さない

7. ディレクトリ要件（生成物）

例：

/doc/requirements.md（この仕様書）

/image/（画像格納）

/src/pages/（ページ）

/src/components/（Header/Footer等）

8. 受け入れ条件（チェックリスト）

 基本色が緑系で、全体が高級感のある見た目になっている

 Top / Access(Map) / Pricing / Staff / Reservation(LP) の5ページがある

 全ページにヘッダーのメニューがある

 Topのヒーロー直下付近に「予約ページへのボタン」がある

 料金体系に4メニュー（フット/タイ/オイル/ヘッド）があり、時間サンプルが入っている

 「現金のみ」がTop・Pricing・Reservationで明確に表示されている

 画像が /image フォルダに格納され、ページから参照されている
